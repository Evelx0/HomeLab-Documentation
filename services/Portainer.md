## Portainer

**Purpose:**<br>
Another user-friendly tool to configure, deploy, manage, etc docker containers. 

**Why Implemented:**<br>
Enables management of more complex infrastructure without the need to e.g write code or use terminal commands.

**Dependencies:**<br>

- Docker
- Ports
- SSL

**Key concepts learned:**<br>

- GUIs can save a lot of time if designed properly.
- HTTPS
- Container deployment

**Troubleshooting:**<br>

- SSL validation
- DNS issues
- Container connectivity
