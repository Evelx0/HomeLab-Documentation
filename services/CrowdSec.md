# CrowdSec

## Purpose

CrowdSec is an open-source intrusion prevention system that analyses logs from services and identifies malicious activity.

It acts as a detection layer, identifying potentially malicious IP addresses and generating remedy actions based on observed behaviour.

---

## Why I Implemented It

As part of my home lab, I wanted to improve the security of externally accessible services (mainly vaultwarden) and gain practical experience with modern security monitoring tools.

CrowdSec was implemented to:

* Monitor logs from internet-facing services.
* Detect malicious activity and attack patterns.
* Provide visibility into attempted attacks against the environment.
* Improve understanding of log analysis and security event monitoring.
* Learn how security tooling can integrate with reverse proxies and exposed services.

---

## Architecture Integration

CrowdSec operates as a monitoring and analysis platform within the home lab environment.

Traffic flow:

Internet
↓

↓
Home Router Port Forwarding
↓
Nginx Proxy Manager
↓
Vaultwarden

Security monitoring:

Internet
↓
Cowrie Honeypot
↓
CrowdSec Log Analysis
↓
Threat Detection & Alerting

CrowdSec analyses logs generated by services and identifies suspicious behaviour based on community-maintained detection scenarios.

---

## Integration with Cowrie Honeypot

Crowrie is deployed as an SSH honeypot to intentionally attract automated scanners and malicious login attempts.

The purpose of Cowrie is not to provide access but to capture information about attack activity.

Cowrie records:

- Failed login attempts
- Username enumeration attempts
- Password guessing attacks
- Automated bot activity
- Common attacker behaviour

CrowdSec consumes and analyses these logs to:

- Identify brute-force patterns
- Detect repeated attack attempts
- Classify malicious IP addresses
- Provide visibility into real-world attack activity

This integration provides practical experience with:

- Security monitoring
- Log analysis
- Threat detection
- Honeypot deployment
- Intrusion prevention concepts

## Key Concepts Learned

### Log Analysis

Most security tooling relies heavily on log data.

Through CrowdSec I gained experience reviewing:

- Authentication logs
- Web server logs
- Docker container logs
- Honeypot event logs

to identify potential issues and suspicious activity.

### Threat Detection

CrowdSec uses predefined detection scenarios to identify behaviour associated with malicious activity.

Examples include:

- Brute-force attacks
- Credential stuffing
- Excessive auth failures
- Automated scanning activity

### Security Monitoring

Deploying CrowdSec improved my understanding of how organisations monitor environments for suspicious behaviour and respond to security events.

### Layered Security

Security controls are more effective when multiple layers work together.

Within the home lab:

- Nginx Proxy Manager provides controlled access to services.
- SSL certificates secure communications.
- WireGuard provides remote private access.
- CrowdSec provides monitoring and threat detection.
- Cowrie provides visibility into attack behaviour.

## Troubleshooting & Operational Experience

### CrowdSec Not Receiving Logs

Issue:
No events appearing within CrowdSec.

Resolution:

- Verified container status using Docker.
- Confirmed log file locations.
- Checked volume mappings.
- Reviewed CrowdSec container logs.

Outcome:
Log ingestion restored successfully.

---

### False Positives During Testing

Issue:
Legitimate testing activity generated detection events.

Resolution:

- Reviewed triggered detection scenarios.
- Validated event sources.
- Adjusted testing methodology.

Outcome:
Improved understanding of detection tuning and event interpretation.

---

## Lessons Learned

- Logs are one of the most valuable troubleshooting and security resources available.
- Most attack activity is highly automated.
- Security monitoring is most effective when integrated with multiple data sources.
- Visibility is often more important than prevention when learning how attacks occur.
- Understanding attacker behaviour improves defensive decision-making.
